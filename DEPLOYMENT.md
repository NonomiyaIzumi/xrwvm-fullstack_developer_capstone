# Deploying this project (free tier, no IBM Cloud / Docker CLI needed)

This repo ships a [`render.yaml`](render.yaml) Blueprint that deploys all three
services (Django web app, dealers/reviews microservice, sentiment analyzer) on
[Render](https://render.com)'s free tier in one shot — no credit card, no CLI
required on your end.

Why Render instead of IBM Cloud Code Engine: the environment this project was
built in has no `ibmcloud` CLI, no Docker daemon, and no cloud credentials, so
Render was chosen as a reachable substitute that still gives a real public
HTTPS deployment URL. If you'd rather deploy to IBM Cloud Code Engine, the
three `Dockerfile`s in `server/`, `database/`, and
`server/djangoapp/microservices/` are the same images `ibmcloud ce app create
--build-source` would use.

## Steps

1. Go to <https://dashboard.render.com/blueprints> and sign in (GitHub OAuth is
   the fastest option).
2. Click **New Blueprint Instance**, pick the
   `NonomiyaIzumi/xrwvm-fullstack_developer_capstone` repo, and confirm — Render
   reads `render.yaml` and proposes all three services
   (`dealership-web`, `dealership-database`, `dealership-sentiment-analyzer`)
   on the Free plan.
3. Click **Apply**. Render builds and deploys all three Docker images. This
   takes a few minutes the first time.
4. Once `dealership-web` shows **Live**, open its URL
   (`https://dealership-web-XXXX.onrender.com`) — that's the app.
5. **One manual follow-up step required:** open the `dealership-web` service
   → **Environment**, set `CSRF_TRUSTED_ORIGINS` to that exact URL (comma
   separate if you add more), and save — this triggers a redeploy. Without
   this, Django admin login will fail CSRF verification behind Render's proxy.
6. The admin password was auto-generated by Render (`DJANGO_SUPERUSER_PASSWORD`
   env var, `generateValue: true`) — copy it from the `dealership-web` service's
   **Environment** tab to log into `/admin/` as `admin`.

## Notes on how this stays stateless-friendly

- The dealers/reviews microservice boots a real, ephemeral, in-process
  MongoDB (`mongodb-memory-server`) and reseeds the demo dealers/reviews on
  every start — no MongoDB Atlas account needed. Reviews posted during a
  session persist until Render spins the free service down for inactivity;
  the next cold start reseeds the same demo data.
- The Django app's SQLite database is likewise ephemeral on Render's free
  tier (no persistent disk). `server/entrypoint.sh` runs migrations, reseeds
  `CarMake`/`CarModel`, and recreates the `admin` superuser on every
  container start, so the app is always in a known-good state after a
  cold start.
